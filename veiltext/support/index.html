Quick tip: Most issues can be resolved by checking the steps below. If you're still stuck, open a GitHub issue and we'll look into it.

Troubleshooting

If Veiltext isn't behaving as expected, try these steps first:

  1. Restart the app — close it fully and reopen.
  2. Check storage permissions — Veiltext needs access to on-device storage to save notes. Go to your device settings and make sure storage permission is granted.
  3. Check available storage — If your device is nearly full, Veiltext may not be able to save new notes.
  4. Update the app — Make sure you have the latest version installed from the App Store or Google Play.
  5. Reinstall — As a last resort, uninstalling and reinstalling the app can clear any corrupted state. Note: this will delete any locally stored notes that have not been backed up.

Frequently Asked Questions

Are my notes backed up to the cloud?
No. Veiltext stores all notes locally on your device. There is no server-side sync or cloud backup. Your notes are yours alone.

Can I recover a deleted note?
Deleted notes cannot be recovered once removed. Veiltext does not maintain a trash or undo history for deletions.

Is Veiltext encrypted?
Yes. Notes are encrypted on-device. The encryption key never leaves your device.

Why can't I open the app after a system update?
Try a clean reinstall. If the issue persists, please report it via GitHub Issues (link below).

Reporting an Issue

If you've found a bug or unexpected behavior, please open an issue on GitHub so it can be tracked and fixed:

When reporting, please include:

  • A clear description of the problem
  • Steps to reproduce the issue
  • Your device model and operating system version
  • The version of Veiltext you're using
  • Any relevant screenshots or error messages
